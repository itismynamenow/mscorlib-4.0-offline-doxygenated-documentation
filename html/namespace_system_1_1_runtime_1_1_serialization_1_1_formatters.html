◆ FormatterAssemblyStyle

enum System.Runtime.Serialization.Formatters.FormatterAssemblyStyle
strong

Indicates the method that will be used during deserialization for locating and loading assemblies.

Enumerator
Simple 

In simple mode, the assembly used during deserialization need not match exactly the assembly used during serialization. Specifically, the version numbers need not match as the Overload:System.Reflection.Assembly.LoadWithPartialName method is used to load the assembly.

Full 

In full mode, the assembly used during deserialization must match exactly the assembly used during serialization. The Overload:System.Reflection.Assembly.Load method of the T:System.Reflection.Assembly class is used to load the assembly.

◆ FormatterTypeStyle

enum System.Runtime.Serialization.Formatters.FormatterTypeStyle
strong

Indicates the format in which type descriptions are laid out in the serialized stream.

Enumerator
TypesWhenNeeded 

Indicates that types can be stated only for arrays of objects, object members of type T:System.Object, and T:System.Runtime.Serialization.ISerializable non-primitive value types.

TypesAlways 

Indicates that types can be given to all object members and T:System.Runtime.Serialization.ISerializable object members.

XsdString 

Indicates that strings can be given in the XSD format rather than SOAP. No string IDs are transmitted.

strong

Specifies the level of automatic deserialization for .NET Framework remoting.

Enumerator
Low 

The low deserialization level for .NET Framework remoting. It supports types associated with basic remoting functionality.

Full 

The full deserialization level for .NET Framework remoting. It supports all types that remoting supports in all situations.
